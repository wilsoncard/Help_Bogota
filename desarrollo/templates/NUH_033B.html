Franquicias

Mediante esta función se habilita el mantenimiento de la tabla en la cual se almacenan los códigos de las diferentes marcas o franquicias existentes en el mercado local, señalando cuáles de éstas se encuentran habilitadas dentro del Sistema por entidad.

El formulario contiene las opciones Actualizar, Eliminar, Adicionar y Detalle. Adicionalmente, cuenta con los hipervínculos Clases de prefijos, Parámetros operativos, Parámetros acciones para el exception file, Parámetros acciones por bloqueos, y Parámetros regiones para el exception file que contienen parametrización adicional para las franquicias.

Adicionar: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguientes bloques:

Franquicias: Conjunto de bloques que permiten definir las principales particularidades de cada franquicia.

Marca

Campo numérico de dos dígitos, obligatorio, asignado a cada una de las marcas o franquicias con las cuales operará la entidad.

Nombre

En este campo alfanumérico de 30 posiciones, obligatorio, se registra la descripción asociada al código de cada franquicia.

Marca habilitada

Campo de chequeo que permite señalar si dicha franquicia se encuentra habilitada o no dentro del sistema, para la entidad.

Establecimiento impuestos

Este campo admite cualquier dato alfanumérico de máximo 11 caracteres. Permite mayúsculas y minúsculas.

El campo Código establecimiento debe validar el código ingresado frente a los establecimientos existentes actualmente en la base de datos para la marca y empresa en la cual se encuentra ubicado el usuario. Si el código ingresado existe, el sistema debe traer automáticamente la descripción del mismo, de lo contrario debe generar un mensaje de error informando que el establecimiento no existe.

Jerarquía asig. cupos

El campo Jerarquía asignación cupos admite cualquier valor numérico de máximo 2 dígitos. Permite ingresar un valor para definir la prioridad en que los cupos de las tarjetas son asignados según la marca.

Longitudes: Conjunto de datos que indican los valores que por defecto define la franquicia para cada uno de los conceptos contenidos en el bloque y los algoritmos a utilizar para verificar la referencia universal asociada a las transacciones.

Bin

Este campo numérico de dos dígitos, obligatorio, permite señalar la longitud máxima por defecto para el bin o prefijo.

Número tarjeta

Campo numérico de dos dígitos, obligatorio, dentro del que se señala la longitud máxima por defecto que tendrán las tarjetas incluyendo el bin o prefijo.

Referencia universal

Campo numérico de dos dígitos, obligatorio, que contiene la longitud máxima del número que identifica de manera exclusiva a cada transacción dentro del universo de transacciones por franquicia, las cuales establecen las rutinas de armado de la misma.

Dígito chequeo referencia

Campo que posee lista de valores adjunta de la que se debe seleccionar el algoritmo que por defecto debe utilizar el sistema para obtener el dígito de chequeo que hace parte de la referencia universal.

Algoritmo referencia

En este campo con lista de valores adjunta, se debe seleccionar la rutina de aplicación del algoritmo por franquicia que por defecto debe utilizar el sistema para obtener o construir la referencia universal asociada a cada transacción.

Comisiones: Conjunto de parámetros definidos por la franquicia, que permiten establecer las condiciones que regirán por defecto los conceptos aquí contenidos.

Tarjeta débito / Tarjeta crédito

Opción que permite definir de manera excluyente si la cifra relacionada en el campo Valor ubicado a continuación, corresponde a un porcentaje o un valor absoluto.

Valor

Campo numérico de 16 dígitos no obligatorio, en el que se registra acorde a lo definido en el campo anterior, el valor absoluto o porcentual a descontar a manera de comisión a los establecimientos por transacciones realizadas con productos débito o crédito y que aplicará para la marca en especifico.Cuando es porcentual, sólo admite hasta 6 caracteres.

Propinas: Conjunto de parámetros definidos por la franquicia, que permiten establecer las condiciones que regirán por defecto los conceptos aquí contenidos.

Derecho propina

Casilla que indica si determinada franquicia permite la liquidación y cobro de propina.

% Máximo no comisionable

En este campo numérico de 6 posiciones, no obligatorio, se registra el valor porcentual máximo o tope para el cobro de propina acordado por la franquicia.

Tolerancia no comisionable

Campo numérico de 14 enteros y 2 decimales no obligatorio, en el que se registra el valor absoluto máximo acordado entre el emisor y el operador (franquicia) en que puede excederse el monto de comisión cobrado.

Código servicio internacional

Campo numérico de cuatro dígitos, no obligatorio, mediante el que cada franquicia determina el código asociado a cada producto (bin, segmento, y grupo de afinidad) señalando que puede ser utilizado tanto a nivel nacional o local como también internacional.

Código servicio nacional

En este campo numérico de cuatro dígitos, no obligatorio, se registra el código determinado por cada franquicia y que estará asociado a cada producto (bin, segmento, y grupo de afinidad) cuyo uso se restringe al nivel nacional o local.

Franquicia internacional: Conjunto de campos que permiten señalar los códigos que identifican la cobertura de la tarjeta 'Local' o 'Internacional', información que hará parte de los datos contenidos dentro de la banda magnética de cada plástico y que serán asignados a nivel de cada Producto en sus Líneas de crédito.

% Adquirencia nacional

Campo numérico de 6 posiciones enteras y 2 decimales, no obligatorio, en el cual se define el porcentaje aplicable sobre las comisiones de los comprobantes de venta para determinar los valores devengados por la entidad como consecuencia del envío de movimiento de canje nacional.

% Adquirencia internacional

Campo numérico de 6 posiciones enteras y 2 decimales, no obligatorio, en el cual se define el porcentaje aplicable sobre el total de los comprobantes de venta para determinar los valores devengados por la entidad como consecuencia del envío de movimiento de canje internacional.

Tipo liquidación

Campo que posee lista de valores adjunta de la que debe seleccionarse el tipo de liquidación de las comisiones entre Intercambio o Adquirencia si es por adquirencia las comisiones son liquidadas según el modelo anterior (porcentajes entre entidades y franquicias), y si es por Intercambio las comisiones son liquidadas según porcentajes pactados entre entidades y establecimientos.

Actualizar: Si el usuario invoca la opción Actualizar se despliega un nuevo formulario en el cual el único campo no modificable es el Código.

Detalle: Si el usuario invoca la opción Detalle se despliega el siguiente formulario:

Botones especiales

Actualizar propinas de servicio : Botón que permite asignarle la información contenida en estos campos a todos los tipos de servicios y establecimientos que no la posean o sólo a los tipos de servicios y establecimientos iguales al actual.

Usuarios Webservice

Se deben parametrizar los usuarios que van a consumir los servicios web y sus correspondientes contraseñas (se guardan encriptadas en la base de datos con un algoritmo triple des):.

Para poder consumir el servicio web se debe tener en cuenta enviar en la cabecera (Add HTTP Header) el Username y el Password (encriptado), labor que debe parametrizarse en cada uno de los métodos.

El formulario contiene los enlaces Actualizar y Adicionar .

Al activar el enlace Actualizar y Adicionar, el sistema despliega el siguiente formulario:

Descripción de campos

Usuario

Campo obligatorio donde se parametriza el nombre de usuario que va a consumir los servicios web.

Password

Campo obligatorio que permite seguridad al usuario al ingreso de una unica contraseña personal de ingreso seguro.

Confirmar password

Campo obligatorio de confirmación del password de usuario.

Actividades

Mediante esta función se habilita la consulta y mantenimiento de la tabla en la cual se almacenan las diferentes actividades que deben ser realizadas para la gestión de cada uno de los tipos de reclamos existentes, permitiendo definir un tiempo específico para cada una de ellas así como el orden en el cual deben ser ejecutadas. Así mismo, se definen los roles que podrán ejecutar dichas actividades y los requisitos que cada una de ellas tendrá.

El formulario contiene las opciones Actualizar, Eliminar y Adicionar. Adicionalmente, cuenta con un filtro de búsqueda y las opciones Requisitos por actividad y Roles por actividad.

Filtro: Se pueden realizar consultas a través de las siguientes opciones:

Código

Campo numérico de 4 dígitos que contiene el código de la actividad que se desea buscar.

Descripción

En este campo alfanumérico de máximo 60 caracteres, se ingresa el nombre asociado a la actividad que se desee buscar.
Tipo de reclamo

En este campo se selecciona de la lista de valores adjunta, el tipo de reclamo que se desea consultar.

Adicionar: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguientes campos:

Código

Campo numérico de 4 dígitos, obligatorio, que contiene el código asignado a cada actividad y que la identifica dentro del sistema.

Descripción

Campo alfanumérico de 60 posiciones, obligatorio, en el cual se registra el nombre de la actividad.
Tipo de reclamo

En este campo obligatorio, se selecciona de la lista de valores adjunta, el tipo de reclamo al cual corresponde la actividad.
Duración

Campo numérico de 3 dígitos, obligatorio, que contiene la duración expresada en número de horas asignadas para realizar dicha actividad. El sistema valida que la suma del número de horas asignadas a las actividades de un tipo de reclamo específico, no pasen el valor parametrizado en el periodo máximo de ejecución de dicho reclamo.
Actividad depende de

De esta lista de valores se selecciona la actividad que debe preceder a la que se está ingresando. La 'Actividad depende de' debe estar terminada o cancelada antes de poder iniciar la ejecución de la actividad que se está parametrizando.

Este campo sólo puede ser nulo cuando la Actividad ingresada es la primera para un tipo de reclamo, de lo contrario siempre exigirá ser diligenciado con la anterior actividad creada para el mismo reclamo.
Rol

En este campo obligatorio, se selecciona de la lista de valores el rol principal al cual se podrá asignar esta actividad.
.

Actualizar: Si el usuario invoca la opción Actualizar se despliega un nuevo formulario en el cual los únicos campos modificables son la Descripción y la Duración.

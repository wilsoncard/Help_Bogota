Parámetros

Mediante esta función se definen, por producto, las diferentes clases o tipos de solicitud ofrecidas o disponibles para éstos y, las etapas que deben surtir.

Descripción de Campos

Bin

Información que aparece por defecto alimentada de los datos registrados en el módulo de mercadeo en la opción de parámetros (Seg., Grupo, Descripción producto, Marca).

Tipo solicitud

Conjunto de campos que permiten a la entidad definir, mediante lista de valores adjunta, todas y cada una de las distintas solicitudes que pueden efectuarse por cada producto.

Parámetros: Dupla de criterios que serán tenidos en cuenta durante el trámite de las solicitudes de cada producto. Estos valores aplican o deben definirse por cada Tipo de solicitud. 

Consulta derogatoria

Campo que permite definir si para el trámite de las solicitudes se tendrán en cuenta los datos contenidos en la tabla de derogatorio de cédulas los cuales corresponden a aquellos números de identificación a los cuales la entidad debe prestar mayor atención por estar allá registrados, puesto que determinan alguna clase de peligro para la misma. Hay dos opciones a saber: Error o Peligro.En el evento que la entidad determine que la validación que debe realizar es Error, significa que si el número de identificación del cliente que está efectuando la solicitud se encuentra en la tabla derogatoria de cédulas, el sistema detiene el proceso en dicho punto y no continua. Sin embargo, si el parámetro indica Peligro el sistema muestra un mensaje de alerta para que la entidad tome las medidas de precaución necesarias, permitiendo continuar con el proceso.

Evaluar solicitud

Campo obligatorio, el cual contiene tres valores (Módulo externo, Si, No) para que de acuerdo con los procesos de la entidad se determine cuales son las pautas para evaluar las solicitudes.De esta manera se evalúa al cliente y su capacidad de endeudamiento.

EtapasEn este espacio la entidad determina los estadios o estaciones por las cuales deben pasar las diferentes solicitudes por producto. Estos valores aplican o deben definirse por cada Tipo de solicitud. 

Ingreso

Esta etapa siempre estará marcada por defecto y es obligatoria puesto que toda clase de solicitud debe ser ingresada para que se realice su respectivo trámite y quede el registro de ésta en el sistema. 

Verificar referencias

Campo que se marca para que la solicitud entre o no a ésta etapa, si la entidad así lo requiere y se registre el resultado de esta tarea.

Envío a C/R

Campo que se marca para que la solicitud entre o no a esta etapa, si la entidad así lo requiere.

Conceptos de C/R

Campo que se marca para que la solicitud entre a esta etapa, si la entidad así lo requiere y estando allá se pueda registrar el resultado de la misma.

Aprobación

Esta etapa siempre estará marcada por defecto y es obligatoria puesto que toda clase de solicitud debe ser aprobada para que se realice su respectivo trámite.

Generacion plásticos

Campo que se marca para que la solicitud entre a esta etapa, genere el plástico por determinada solicitud si la entidad así lo requiere.

Activar

Campo que se marca para que la solicitud entre a esta etapa, si la entidad así lo requiere.

Generación sobreflex

Campo que se marca para que la solicitud entre a esta etapa, y genere información del número de identificación personal del cliente por determinada solicitud si la entidad así lo requiere.

Ingreso solicitudes nuevas

A través de esta opción se ingresan las solicitudes de nuevo débito. Al invocar la opción, se despliega un formulario en el que se deben digitar los datos de identificación del cliente para quien se desea ingresar la solicitud y activar el botón Buscar. El sistema valida la existencia de un registro que coincida con el tipo y número de identificación.

Descripción de campos

Tipo identificación

Campo tipo combo en el cual se puede seleccionar el tipo de documento de identificación asociado con el cliente a quien se desea ingresar la solicitud.

Número identificación

En este campo alfanumérico de hasta 16 caracteres, obligatorio, se registra el número de identificación del cliente.

Una vez validados los datos de identificación ingresados, se despliega el formulario anterior que, además de contener información básica del cliente, dispone del enlace Solicitudes que, al activarse, ilustra si el cliente ya posee solicitudes de nuevo débito en proceso o no finalizadas. Si no existen esa clase de registros, el formulario se despliega en blanco. Además de los enlaces: Actualizar, Eliminar, Adicionar y Detalle, el formulario dispone del enlace Consultar etapas, el que permite ver el histórico de las etapas por las que ha pasado la solicitud.

Una vez hechas las validaciones respectivas, el sistema despliega un nuevo formulario en el que se deben indicar o registrar, los datos requeridos para el ingreso de la solicitud:

Descripción de campos

Producto

Campo de salida que muestra el bin, producto, grupo de afinidad y descripción del producto débito solicitado por el cliente.

Emite plástico

Este campo es solo de salida y muestra por defecto la opcion siempre en Si.

Nombre realce

Campo de salida que despliega el nombre del cliente y que va a ser impreso o realzado en la tarjeta.

Ciclo de facturación

En este campo de salida se muestra el ciclo de facturación al que pertenece la tarjeta y que será la base para determinar las fechas para el cobro de los costos generados por cuota de manejo.

Solo consulta

Campo obligatorio, con lista de valores tipo combo, de la que puede indicarse Normal, para señalar que la tarjeta no tiene restricciones para su uso o restringida para solo Consulta.

Sucursal

En este campo obligatorio y que contiene lista de valores, se selecciona la sucursal de radicación de la tarjeta débito.

Tipo de cuenta

Campo obligatorio tipo combo en el que se selecciona entre Ahorros, Corriente o Rotativo el tipo de cuenta asociado a la tarjeta débito.

Número de cuenta a debitar

En este campo obligatorio se registra el número de cuenta asociado con la tarjeta débito.

Moneda

En este campo obligatorio con lista de valores, se selecciona la moneda en la cual se encuentra denominada la cuenta asociada con la tarjeta débito.

Agente gestión

Campo no obligatorio, en el que se digita o se selecciona de la lista adjunta, el código del agente asociado a la solicitud del cliente.

Punto de entrega

Campo no obligatorio, en el que, si aplica, se digita o se selecciona de la lista adjunta, el código del punto físico de la dependencia de la entidad en la que le será entregado el plástico de la tarjeta solicitada.

Nombre empresa, Tipo identificación y Número identificación

Conjunto de campos que aplican para cuando se trate de una solicitud para un producto empresarial, el sistema no valida ninguno de esos datos que corresponden a la empresa a la que está vinculado el producto solicitado por el cliente.

Tarjeta Chip

Campo tipo checkbox en el que se indica si la tarjeta a expedir dispone o no de ese componente o dispositivo.

Flag personalizada

Campo tipo checkbox en el que se indica si en el plástico de la tarjeta se le debe adicionar algún distintivo a manera de personalización del mismo. En caso afirmativo, en el campo adjunto se debe digitar la ruta en la que se encuentra el archivo correspondiente.

Sistema de transporte

Campo que aplica si dentro de los atributos de la tarjeta, se tiene que puede ser usada para el pago de la tarifa del servicio de transporte masivo, caso en el cual, se debe digitar o seleccionar el código correspondiente.

Contactless

Campo que se activa cuando el plástico corresponde a una tarjeta chip, posee lista de valores tipo combo, de la que debe seleccionarse entre: Tarjeta, Celular micro SD o Celular simcard, la manera en que se habilita dicho servicio de utilización sin requerirse el contacto físico.

Modelo teléfono

En caso de que el contactless se habilite para el celular, en este campo se indica a cuál modelo corresponde dicho teléfono.

Una vez diligenciada la información suficiente, se activa el botón con el símbolo de palomita, check o visto bueno, al final del formulario, en la parte inferior derecha, el sistema realiza las validaciones a que haya lugar y, de cumplirse con ellas, retorna un mensaje "Proceso finalizado satisfactoriamente", creando el registro.

Actualizar: Al activar ese enlace, se despliega un nuevo formulario, en el cual los únicos campos modificables son: Sucursal, Tipo cuenta, Número cuenta y Sucursal cuenta.

Detalle: Al activar ese enlace se despliega el siguiente formulario, con la información de la solicitud, y en el cual, ninguno de sus campos es modificable:

Consultar etapas : Al activar ese enlace, se despliega un formulario que, en su encabezado ilustra varios datos que identifican: Tipo solicitud, Fecha ingreso, Etapa actual y Producto, además, permite visualizar el detalle de las diferentes etapas, parametrizadas para el tipo de solicitud y por producto, por las cuales ha avanzado la solicitud o aún no lo ha hecho.

Descripción de campos

Etapa

Campo que muestra el código y descripción de cada una de las etapas por las cuales ha avanzado la solicitud.

Forzada

En este campo se indica si el avance de la solicitud en cada etapa fue resultado del proceso normal o si por el contrario la solicitud se avanzó de manera forzada pasando por alto las validaciones propias de la etapa.

Fecha hora inicio

Campo que despliega la fecha y hora en la cual la solicitud arribó a cada una de las etapas.

Fecha hora fin

En este campo se muestra la fecha en la cual la solicitud fue avanzada a una etapa siguiente o terminó su trámite en cada una de las etapas.

Avanzar solicitud: Una vez finalizado el proceso de captura de los datos de la solicitud y mostrarse el mensaje de que el proceso fue satisfactorio, se dispone de este botón especial, el que, al activarse, hace que se surta la primera etapa parametrizada para este tipo de solicitud.

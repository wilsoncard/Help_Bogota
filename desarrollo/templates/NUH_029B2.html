Autorización y restricción

Restricciones: Esta opción permite determinar diferentes clases de restricciones por cada tipo de rol. El sistema permite los siguientes tipos de restricciones: Campo, Tipo de transaccioón, Tipo de solicitud, Novedades, Productos, y Botones.

El formulario contiene los enlaces: Eliminar y Adicionar. Adicionalmente, cuenta con un filtro de búsqueda y un botón en la parte superior que permite volver al formulario principal.

Filtro: Se pueden realizar consultas a través de las siguientes opciones:

Tipo de restricción

Lista de valores que permite filtrar entre una de las siguientes opciones: Nulo, Campo, Tipo de transacción, Tipo de solicitud, Novedades, Productos, o Botones.

Descripción

Campo que debe aceptar hasta 40 caracteres alfanuméricos. Se muestra sin lista de valores y acepta la búsqueda con el símbolo %.

Adicionar: Al activar ese enlace se despliega el siguiente formulario, cuyos campos se habilitan de acuerdo a los seleccionado en la lista de valores Tipo de restricciones a excepción del campo Nulo.

Campo

Este campo muestra un combo con las siguientes opciones (Número de tarjeta y Número de tarjeta débito).

Tipo de transacción

Este campo contiene lista de valores únicamente con las transacciones parametrizadas en la opción de Transacciones por empresa.

Tipo de solicitud

Campo que muestra un combo con las siguientes opciones: Crédito nuevo, Variación cupo crédito, Renovación crédito, Reposición crédito, Reexpedición crédito, Reactivación crédito, Línea crédito adicional crédito, Cambio ciclo facturación crédito, Cancelación voluntaria crédito, Cambio producto crédito, Débito nuevo, Renovación débito, Reposición débito, y Reexpedición débito.

Novedades

Este campo contiene lista de valores únicamente con las transacciones parametrizadas en la opción de Novedades administrativas.

Producto

El campo Producto (Bin, Segmento, Grupo y Descripción), muestra lista de valores acorde con todos los productos Débito y Crédito definidos para la empresa en la cual se encuentra ubicado el actor.

Botones

Campo que contiene lista de valores únicamente con las transacciones parametrizadas en la opción de Botones seguros.

Rol de negocios

El campo Rol de negocios debe permitir ser diligenciado con cualquier dato alfanumérico de máximo 60 caracteres, permitiendo mayúsculas y minúsculas.

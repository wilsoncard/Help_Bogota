Parámetros solicitudes tarjeta débito

Al invocar la opción, se despliega un filtro con el mismo nombre, en el que se debe o digitar los tres datos que componen el producto: Bin, Segmento y Grupo de afinidad o bien, se pueden buscar en la lista activada mediante la lupa a la derecha, incluso por el nombre o descripción del mismo, siendo este el paso previo para que se muestre la data de dichos parámetros por cada producto:

Una vez seleccionado el producto de interés y activando el botón Buscar, si los datos son válidos, se despliega el siguiente formulario, en cuya parte superior se muestra la combinatoria y nombre o descripción del producto, mismos datos del filtro utilizado, más abajo del cual, se dispone de los enlaces: Actualizar, Eliminar, Adicionar y Detalle. En el bloque principal, se ilustran los diferentes tipos de solicitudes que la entidad ha definido que se pueden gestionar por producto:

Tipo solicitud

Campo que muestra el código con el que el sistema identifica a cada una de las distintas solicitudes que se le han asociado al producto.

Descripción solicitud

Campo que ilustra el nombre o descripción del tipo de solicitud asociada al producto.

Ingreso

Esta etapa siempre estará marcada por defecto y es obligatoria puesto que toda clase de solicitud debe ser ingresada para que se realice su respectivo trámite y quede el registro de ésta en el sistema. A partir de este punto, se ilustran las cinco etapas más relevantes de las solicitudes, indicando para cada una, si aplica o no, respecto de cada tipo de solicitud.

Aprobacion

Campo que muestra un Si, siempre y cuando al momento de crearse o modificarse el registro, la entidad indica que se debe surtir dicha etapa, en caso contrario, se muestra el valor No.

Generacion plastico

Campo que muestra un Si, siempre y cuando al momento de crearse o modificarse el registro, la entidad indica que se debe surtir dicha etapa, en caso contrario, se muestra el valor No.

Generacion sobreflex

Campo que muestra un Si, siempre y cuando al momento de crearse o modificarse el registro, la entidad indica que se debe surtir dicha etapa, en caso contrario, se muestra el valor No.

Activar

Campo que muestra un Si, siempre y cuando al momento de crearse o modificarse el registro, la entidad indica que se debe surtir dicha etapa, en caso contrario, se muestra el valor No.

Adicionar: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguientes campos:

Tipo solicitud

Campo obligatorio, posee lista de valores tipo combo, mediante la cual se le permite a la entidad, definir las distintas solicitudes que pueden efectuarse por cada producto.

Consulta derogatoria

Campo que permite definir si para el trámite de las solicitudes, se tendrán en cuenta los datos contenidos en derogatorio de cédulas, los cuales corresponden a aquellos números de identificación a los que la entidad debe prestar mayor atención, por estar allí registrados, puesto que determinan alguna clase de riesgo para la misma. Hay dos opciones a saber: Error o Peligro. En el evento que la entidad determine que la validación que debe realizar es Error, caso en el cual, el sistema detiene el proceso en dicho punto y no continua. Sin embargo, si el parámetro indica Peligro, el sistema muestra un mensaje de alerta para que la entidad tome las medidas de precaución necesarias, permitiendo continuar con el proceso.

Evaluar solicitud

Campo no obligatorio, posee lista tipo combo con los valores Si / No, para que, de acuerdo con los procesos de la entidad, se determine cuáles son las pautas para evaluar las solicitudes. De esta manera se evalúa al cliente y su capacidad de endeudamiento.

Generación sobreflex

Campo no obligatorio, que indica la forma como se debe generar el sobreflex de clave durante el proceso de la solicitud. Los posibles valores son: Nulo, Normal o Después.

"Etapas"

Conjunto de campos, en los que, frente a cada uno, se cuenta con un checkbox, que permiten señalar cuáles etapas deberá surtir el tipo de solicitud que se está adicionando.

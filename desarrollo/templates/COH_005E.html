Datos del mensaje

Esta función permite establecer los datos a incluir en los mensajes que se intercambian entre el equipo central autorizador y los adquirentes, que están definidos en el sistema.

Estos datos corresponden al estándar del protocolo universal BASE24 definido por las franquicias Visa y Master Card para este efecto, y sus características son usadas por este módulo en el proceso de interpretación de mensajes y en la solución de los mismos.

Filtro: El formulario cuenta con un filtro inicial de búsqueda, necesario para la consulta de la información:

Marca

Campo de salida que a partir del bin despliega la marca o franquicia a la cual pertenece el producto.

Código

Campo numérico de 3 digitos, obligatorio en el que se registra el código de operación.

Tipo de dato

A través de una lista de valores, la entidad define las características del dato asociado con cada uno de los mensajes que va a viajar por el sistema durante el proceso de intercambio entre el equipo central autorizador y los adquirentes.

El formulario contiene las opciones Actualizar, Eliminar, Adicionar y Detalle:

Adicionar: Si el usuario invoca la opción Adicionar se despliega un formulario con los siguiente campos:

Marca

Campo de salida que a partir del bin despliega la marca o franquicia a la cual pertenece el producto.
Código

Campo numérico de 3 digitos, obligatorio en el que se registra el código de operación.

Descripción

Campo a la explicación del código definido en el campo anterior.

Tipo de dato

A través de una lista de valores, la entidad define las características del dato asociado con cada uno de los mensajes que va a viajar por el sistema durante el proceso de intercambio entre el equipo central autorizador y los adquirentes.

Atributo longitud

Campo numérico que indica si la longitud es fija o variarible de dos a tres posiciones.

Máxima longitud

Campo numérico de 3 digitos en el que se registra la longitud máxima.
Tipo de dato viaja

Campo numérico de 3 digitos en el que se registra el tipo de dato enviado.

ATM

En este campo se define la longitud del dato para las operaciones en ATM's.

POS

En este campo se define la longitud del dato para las operaciones en POS's.

Actualizar: Si el usuario invoca la opción Actualizar se despliega el mismo formulario de adición, pero con los campos marca y código en segundo plano porque no se pueden actualizar.

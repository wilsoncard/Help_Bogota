Registro de autorizaciones

Mediante esta función se habilita un centro de autorizaciones manual, con el fin de atender los establecimientos que posean puntos de venta (P.O.S) conectados al sistema, o simplemente para suplir las funciones del módulo de autorizaciones en línea, cuando por alguna razón esté fuera de servicio. Las únicas autorizaciones posibles de registrar en esta forma son aquellas que deban darse a transacciones de ventas o avances.

Datos autorización: El sistema verifica que todos los campos contengan datos, por ser obligatorios, luego de lo cual, permite avanzar mediante el botón especial Cargar información.:

Número tarjeta

Campo obligatorio en el que se debe ingresar el número de la tarjeta del cliente para la que se gestiona la autorización. Al momento de activar el botón especial Cargar información, el sistema valida que el dato ingresado exista, si el número es errado, retorna el mensaje correspondiente.

Valor transacción

Campo obligatorio, que en un máximo de doce enteros y dos decimales, debe contener el monto de la autorización que se está gestionando.

Transacción

Campo obligatorio en el que se puede digitar o seleccionar de la lista de valores provista, y poblada previamente en la opción orígenes de transacciones para el origen 8 AUTORIZACIONES NORMALES, en la ruta Núcleo / Configuración de parámetros / Parámetros de operación, el código de la clase de transacción para la que se gestiona la autorización. Si el código digitado no hace parte de la lista, el sistema notifica que ese dato no se encuentra.

Establecimiento

Campo obligatorio en el que se puede digitar o seleccionar de la lista de valores provista, y poblada previamente en la opción Información de establecimientos, en la ruta Núcleo / Información cliente, el código del comercio en el que se va a efectuar la transacción para la que se gestiona la autorización. Si el código digitado no hace parte de la lista, el sistema notifica que ese dato no se encuentra. Si el código existe, se despliega el nombre del mismo.

Moneda

Moneda Campo obligatorio en el que se puede digitar o seleccionar de la lista de valores provista, y poblada previamente en la opción monedas en la ruta Núcleo / Configuración de parámetros / Parámetros generales, el código de la moneda en que se denomina la transacción para la que se gestiona la autorización. Si el código digitado no hace parte de la lista, el sistema notifica que ese dato no se encuentra. Si el código existe, se despliega el nombre de la misma.

Al cargar la información a través del botón especial, se muestra un formulario que contiene un enlace al formulario Estadísticas, tres bloques de datos, y los botones Proceso Autorización y Cancelar.

Número tarjeta

Muestra el número de tarjeta enmascarado (si aplica), para la que se está gestionando la autorización.

Nombre plástico

Muestra el nombre del cliente como aparece en el plástico de la tarjeta.

Tipo identificación

Contiene la abreviatura y nombre del tipo de documento de identificación del tarjetahabiente.

Número identificación

Contiene el número del documento de identificación del tarjetahabiente.

Fecha vencimiento

Muestra la fecha de vencimiento, enmascarada (si aplica), de la tarjeta para la que se está gestionando la autorización.

Transacción

Contiene el código y nombre de la transacción para la que se está gestionando la autorización.

Moneda

Contiene el código y nombre de la transacción en que se denomina la transacción para la que se gestiona la autorización.

Valor transacción

Campo que muestra el monto de la autorización que se está gestionando.

Establecimiento

Campo que muestra el código y nombre comercial del establecimiento en el que se va a efectuar la transacción para la que se gestiona la autorización.

Horario

Campo que contiene información del horario en que el establecimiento está autorizado para operar normalmente; este dato ayuda a la entidad a detectar comportamientos fraudulentos con el uso de la tarjeta.

Número identificación

Campo que ilustra el número de identificación del establecimiento en donde se está realizando la transacción.

Razón social

Muestra el nombre con el cual se encuentra registrado el establecimiento ante la Cámara de Comercio o la entidad que cumpla esa función.

Servicio

Despliega el código y nombre del servicio asociado al comercio a través del cual se realiza la utilización para la que se solicita la autorización.

Convenio

Campo de salida que muestra el código y el nombre del convenio al que pertenece la tarjeta, aplica para cuando la misma es de marca privada.

Proceso autorización: Al activar este botón especial, el sistema realiza las validaciones correspondientes y genera la respuesta a la misma.

En este bloque Información de la Autorización, se muestra la respuesta del sistema a la solicitud de la autorización.

Código respuesta

Muestra tanto el código como la descripción de la respuesta dada por el sistema a la solicitud de autorización.

Número autorización

En caso de que la solicitud de autorización haya sido exitosa, en este campo se muestra el número, diferentes a ceros, que le fue asignado por el sistema. En otros casos se muestran solo ceros.

Estadísticas: Permite conocer las utilizaciones realizadas a través de la línea de crédito a una fecha determinada.

Detalle: A través de este botón el sistema inicia el proceso de generar la autorización, y muestra el resultado en un nuevo bloque que se muestra en la parte inferior del formulario: Información de la autorización.

Línea crédito

Contiene el nombre o descripción de la línea asociada al código anterior.

Moneda

Despliega el nombre de la moneda en que se denomina la respectiva línea de crédito.

Disponible línea

Muestra el cupo disponible de la línea de crédito de acuerdo con el producto.

Cantidad

Indica el número de autorizaciones que aún no han sido procesadas.

Valor

Señala el monto total de las autorizaciones que aún no han sido procesadas.

Semana

A través de estos campos la entidad conoce la cantidad de autorizaciones realizadas de acuerdo con la frecuencia que se muestra: Corresponde al total de utilizaciones que han requerido de autorización durante el día.

Día

Despliega el total de utilizaciones que han requerido de autorización durante la semana.

Día

En este grupo de información la entidad conoce la veces en las cuales se han solicitado autorizaciones y éstas no han sido procesadas: Informa el monto total utilizado durante el día.

Semana

Registra el monto total utilizado durante la semana.
